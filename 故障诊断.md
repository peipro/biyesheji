# 冷水机组性能诊断方案

## 现状

- ✅ 已有**14843条正常运行数据**（1分钟间隔，连续监测约36天）
- ✅ 已完成**8特征+卡尔曼滤波**数据预处理管线
- ❌ **缺少故障标签数据**：未采集到实际故障样本

无标签情况下，性能诊断转化为**无监督异常检测**问题。

---

## 方法一：原始VAE（基线方法）

### 原理

标准VAE：编码器输入9维（8特征+COP）→ 潜在空间 → 解码器重构9维。诊断基于**所有维度的平均重构误差**。

### 问题

**检测的是"运行状态偏离主流模式"而非"性能退化"。**

当设备低频运行时（泵频降低），特征模式偏离训练数据主流→总重构误差升高→标记为异常。但此时COP可能完全正常甚至更高。实验结果证实：95%阈值标记的"异常"样本COP反而比正常样本高19.62%。

### COP-误差关联：r = -0.023（近乎无关）

---

## 方法二：双头VAE（改进方法）✅

### 架构改进

针对原始VAE的问题，修改架构：

```
原始VAE：
  Encoder: [8特征, COP] → z → Decoder: z → [8特征, COP]
  ↑ COP参与编码="偷看答案"
  诊断：9维平均误差 → 检测"任何模式偏离"

双头VAE：
  Encoder: [8个物理特征] → z → Decoder_feat: z → 特征重构(8维)
                                  → Decoder_COP: z → COP预测(1维)
  ↑ COP不参与编码，须从特征推断
  诊断：COP预测误差 → 检测"COP是否低于预期"
```

关键设计：**编码器只看物理特征，COP由独立的解码头从潜在空间预测**。强迫模型学习"给定工况下COP应该多少"。

### 损失函数

```
Loss = MSE(8个物理特征) + 5 × MSE(COP) + β × KL
```

COP权重更高，确保潜在空间保留COP相关信息。

### 诊断逻辑

诊断不使用总重构误差，而是**只关注COP预测的负残差**（实际COP < 预测COP = 性能退化）：

| 信号 | 含义 | 诊断 |
|:----|------|:----:|
| COP残差 ≈ 0 | 实际COP与预期一致 | ✅ 正常 |
| COP残差 > 0 | COP高于预期 | ✅ 高效运行 |
| **COP残差 << 0** | **COP显著低于预期** | **⚠️ 性能退化** |

### 阈值设定

| 阈值 | 含义 | 理论精度 |
|:----|:----|:--------:|
| 残差5%分位 | COP低于95%的预期情况 | ≥95% |
| 残差1%分位 | COP低于99%的预期情况 | ≥99% |
| 持续≥5分钟过滤 | 孤立噪声不报警 | >99.9% |

## 实验结果

### 双头VAE训练

| 指标 | 数值 |
|------|------|
| 训练数据 | data_feature_engineered_v5_kalman_v8.xlsx |
| 样本数 | 14,843 |
| 特征重构MSE（测试集） | 0.0168 |
| COP预测MSE（测试集） | 0.3317 |
| COP残差均值 | -0.025 |
| COP残差标准差 | 0.551 |

### COP-残差关联

| 指标 | 原始VAE | **双头VAE** |
|:----|:-------:|:-----------:|
| COP-残差相关系数 | **-0.023**（无关） | **+0.6327**（强正相关） |
| 最低5% COP命中率 | 7.7% | **23.1%** |

**正相关意味着：COP越低 → 负残差越大 → 正确标记为退化。** 这是诊断有效的核心证据。

### 退化检测结果

| 阈值 | 检测数 | 占比 | 退化样本平均COP |
|:----|:-----:|:---:|:--------------:|
| 残差5%分位 | 743 | 5.01% | **1.1769** |
| 残差1%分位 | 149 | 1.00% | **0.5467** |
| 改进Z-score | 544 | 3.67% | 1.0162 |

正常COP均值=2.30，退化样本COP均值=1.18（降低49%），说明检测到的确实是低COP时段。

### 最长退化时段

| 排名 | 时长 | 时间范围 | 平均COP | 最小COP | 平均残差 |
|:---:|:---:|:---------|:-------:|:-------:|:--------:|
| 1 | 31分钟 | 2025-05-06 06:43~07:14 | 1.9122 | 0.5262 | -1.4210 |
| 2 | 21分钟 | 2025-05-04 08:06~08:27 | 1.3598 | 0.0138 | -1.0499 |
| 3 | 19分钟 | 2025-05-04 11:27~11:48 | 0.6586 | 0.0201 | -1.7493 |

### 正常 vs 退化的特征对比

| 特征 | 正常均值 | 退化均值 | 偏差 |
|:----|:-------:|:--------:|:----:|
| temperature | 21.41 | 22.99 | +7.38% |
| humidity | 41.42 | 45.02 | +8.68% |
| temp_diff | 1.59 | 2.50 | **+57.35%** |
| lxj_evap_press_avg | 2.70 | 2.83 | +4.65% |
| lxj_cond_press_avg | 5.57 | 5.03 | -9.69% |
| A4冷冻泵_f | 24.68 | 47.29 | **+91.61%** |
| **system_cop** | **2.30** | **1.18** | **-48.92%** |

物理含义：退化时段系统出力增大（温差+57%、泵频+92%），但COP反而降低49%，说明系统效率明显下降。

### 双向检测 vs 退化检测对比

| 类别 | 样本数 | 平均COP | 说明 |
|:----|:-----:|:-------:|:-----|
| 两者一致 | 197 | — | 极端异常 |
| **仅退化标记** | **546** | **1.38** | 退化检测正确识别，双向法漏报 |
| 仅双向标记 | 546 | 4.75 | 高COP样本被误报为异常 |

**关键结论：原始VAE使用的双向误差（总重构误差）会漏报低COP退化样本，同时误报高COP样本。** 双头VAE的退化检测方案解决了这个问题。

## 方法与脚本

### 文件

| 文件 | 说明 |
|:----|:-----|
| `vae_dualhead_train.py` | 双头VAE训练：编码器仅编码8维特征，解码器拆为特征头+COP头 |
| `vae_dualhead_diagnosis.py` | 诊断：基于COP预测负残差进行退化检测 |
| `vae_dualhead_best.pth` | 训练好的模型权重 |
| `vae_dualhead_scaler.pkl` | StandardScaler归一化参数 |

### 运行命令

```bash
# 训练
python vae_dualhead_train.py --input data_feature_engineered_v5_kalman_v8.xlsx

# 诊断
python vae_dualhead_diagnosis.py --input data_feature_engineered_v5_kalman_v8.xlsx
```

## 扩展方向

- **时序VAE（LSTM-VAE）**：将编码器/解码器改为LSTM，捕捉渐变式性能退化
- **多模型投票**：结合双头VAE与简单统计阈值进行联合诊断
- **在线诊断**：将模型部署为滑动窗口检测，实时输出退化概率
