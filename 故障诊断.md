# 冷水机组故障诊断研究方案

## 当前现状

- ✅ 已有大量**正常运行数据**（约14000分钟，10天连续监测）
- ❌ **缺少故障标签数据**：没有采集到实际故障样本

在无标签情况下，故障诊断任务转化为**异常检测**问题：学习正常数据的模式，识别偏离正常模式的样本判定为故障。

---

## 方案一：基于重构误差的无监督异常检测（推荐）

### 方法原理

训练一个自编码器（AutoEncoder）或变分自编码器（VAE），只学习正常数据的重构。故障样本由于从未见过，重构误差会显著高于正常样本，据此判定故障。

### 网络结构

```
输入：9维特征（和COP预测相同特征）
  ↓
编码器：9 → 32 → 16 → 8（潜在空间）
  ↓
解码器：8 → 16 → 32 → 9（重构输入）
输出：重构的特征向量
```

**训练目标**：最小化重构误差（MSE）

**推理**：对新样本计算重构误差，误差超过阈值 → 判定为故障。

### 阈值确定

- 在验证集（正常数据）上确定阈值
- 选择阈值使得正常样本被误判为故障的概率控制在一定范围（如5%）

### 优点

- ✅ 不需要故障标签，只需要正常数据
- ✅ 能检测多种未知故障
- ✅ 网络结构简单，训练快
- ✅ 特征和COP预测共用，数据可以复用

### 变种：VAE（变分自编码器）

不仅使用重构误差，还可以利用潜在空间的概率密度，概率低于阈值判定为异常，性能通常比AE更好。

---

## 方案二：基于预测残差的异常检测

### 方法原理

利用已经训练好的**COP预测模型**：
- 正常工况：模型预测准确 → 残差小
- 故障工况：传感器偏移或机组故障 → 预测偏离实际 → 残差大

残差超过阈值则判定为故障。

### 实现步骤

1. 使用已有数据集训练好COP预测模型（推荐XGBoost或LSTM）
2. 在验证集上统计残差分布
3. 确定残差阈值（如残差 > 均值 + 3倍标准差）
4. 在线运行时，实时计算残差，超过阈值报警

### 优点

- ✅ 可以复用现有的预测模型和数据，不需要额外训练
- ✅ 故障和性能退化直接相关，COP预测残差本身就反映了工况异常
- ✅ 同时完成性能预测和故障诊断两个任务

### 缺点

- 只能检测导致COP预测偏差的故障，对不影响COP的传感器故障敏感
- 阈值选择需要经验

## 方案三：传统无监督异常检测

### Isolation Forest（孤立森林）

- 原理：随机切分数据，异常点更容易被提早切分出来，路径长度更短
- 实现：直接调用sklearn，简单快捷
- 优点：不用深度学习，训练快，参数少

### One-Class SVM

- 原理：在核空间学习正常数据的边界，边界外判定为异常
- 对高维小样本效果较好
- 缺点：大数据集训练慢

### 适用场景

- 快速 baseline 对比
- 特征维度低（本问题只有9维），非常适合传统方法

## 方案四：基于物理规则的故障诊断

### 方法原理

不需要机器学习，直接基于冷水机组运行物理原理和经验阈值进行故障判定。

### 常见故障检测规则

| 故障类型 | 检测规则 |
|----------|----------|
| 传感器漂移 | 传感器读数超出正常范围 → 报警 |
| 冷凝器结垢 | 冷凝压力偏高，COP偏低 → 报警 |
| 蒸发器脏污 | 蒸发压力偏低，COP偏低 → 报警 |
| 冷冻泵故障 | 频率很高但温差很小 → 矛盾，报警 |
| 冷却塔故障 | 冷凝温度偏高 → 报警 |
| 机组过载 | 功率过高，COP异常低 → 报警 |

### 优点

- ✅ 完全不需要数据标签
- ✅ 可解释性强，符合运维人员思维
- ✅ 可以和机器学习方法结合形成混合方案

### 缺点

- 需要领域专家知识
- 无法检测未知类型故障
- 阈值需要经验调试

---

## 方案五：注入模拟故障扩展数据集

### 方法原理

在现有正常数据基础上，人工注入常见故障，生成带标签的故障数据集，转化为有监督学习问题。

### 常见故障注入方式

| 故障类型 | 注入方法 |
|----------|----------|
| 传感器漂移 | 对传感器读数加固定偏差：`x' = x × (1 + ε) + μ` |
| 传感器偏差 | 完全固定在某个错误值 |
| 传感器噪声 | 增加额外高斯噪声 |
| 冷凝器结垢 | 人为提高冷凝压力，降低COP |
| 蒸发器脏污 | 人为降低蒸发压力 |
| 部分负荷异常 | 改变运行台数和负载对应关系 |

### 优点

- ✅ 可以得到带标签数据集
- ✅ 可以控制故障严重程度，做定量研究
- ✅ 可以直接使用有监督故障诊断算法

### 缺点

- 模拟故障和实际故障可能有偏差
- 需要对故障机理有足够了解

## 方案六：使用公开冷水机组故障数据集

### 可用公开数据集

**RWTH Aachen University 冷水机组故障数据集**
- 包含：7种不同故障，4种故障严重程度
- 正常 + 故障共约5万样本
- 可以直接下载用于预训练或迁移学习

网址：https://arxiv.org/abs/1910.00696

### 研究思路

1. 在公开故障数据集上训练有监督故障诊断模型
2. 利用本研究的正常数据进行微调
3. 迁移到实际场景

### 优点

- ✅ 有完整标签，可以直接做有监督学习
- ✅ 是学术界常用基准，方便对比

## 实施建议

### 推荐路线图（循序渐进）

**阶段一**：快速实现基线
- 实现传统方法：Isolation Forest 做异常检测
- 实现基于预测残差的方法（复用现有COP预测模型）

**阶段二**：深度学习方法
- 训练VAE变分自编码器，基于重构误差检测
- 和传统方法对比性能

**阶段三**：结合物理规则
- 添加基于物理知识的规则判断
- 形成机器学习 + 物理规则的混合诊断系统

**阶段四**（可选）：
- 注入模拟故障，训练有监督分类器
- 或引入公开数据集做迁移学习

## 模型评估方法（无标签情况下）

如果完全没有真实故障标签，可以从以下角度评估：

1. **已知正常样本的误报率**：在测试集正常数据上，统计被判定为异常的比例，误报率越低越好
2. **可解释性**：检测出的异常样本在物理上是否合理
3. **阈值鲁棒性**：阈值在一定范围内变化时，结果是否稳定

如果后续采集到少量真实故障样本，可以：
- 计算Precision、Recall、F1分数
- 绘制ROC曲线，计算AUC
